# Plans And Proposals

Use this directory for future-facing or conditional guidance that is not part of
the current implemented architecture.

## What Belongs Here

- recommendations that depend on external approvals or prerequisites
- proposed simplifications that may never be implemented
- deployment or platform directions that are not yet the canonical runtime model

## What Does Not Belong Here

- current system behavior
- active coding patterns and guardrails
- implementation guidance that must be followed today

Those belong in:

- [`docs/architecture/`](../architecture/)
- [`docs/patterns/`](../patterns/)
- [`docs/frontend/`](../frontend/)
- [`docs/testing/`](../testing/)

## Lifecycle

If a plan becomes the implemented standard, move the durable guidance back into
the canonical docs set and update cross-links.
